# ① 残業上限チェック（36系）— サンプル

## これは何か
顧問先「株式会社サンプル製造」の残業CSVを貼り、上限系の一次判定を出すデモ。

## シート構成
| タブ | 内容 |
|------|------|
| `勤怠_残業` | `data/勤怠_残業.csv` を貼付 |
| `判定結果` | メニュー実行で自動生成 |

## セットアップ
1. 新規スプレッドシートを作成
2. シート名を `勤怠_残業` にし、CSVを貼る
3. Apps Script に `Code.gs` を貼って保存
4. 再読み込み → **ロジつく労務 > 上限チェックを実行**

### すぐ見る（Google不要）
- ブラウザで `demo.html` を開く（サンプル判定が即表示）

### Web UI（見栄え版）
- URL: `DEMO.md` 参照（GAS Webアプリ）
- シートメニュー **ロジつく労務 → 画面を開く** でも同じUI

### Google上にデモを作る（要ログイン）
```bash
cd samples/uplink-sharoushi/01-overtime-36
./create-google-demo.sh
```
初回はブラウザでGoogle許可が必要。完了後、共有リンクを「閲覧可」にすればヒアリング用URLになる。

## ヒアリングで聞くこと
- 今の残業実績、どのファイル／列名でもらっているか
- 特別条項の回数管理をどう見ているか
- 職員に一次チェックを任せたいか
- このシートを先生のGoogleに置いて一緒に直す形でよいか
